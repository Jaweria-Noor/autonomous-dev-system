## Requirements Document: Responsive Login Page

### Functional Requirements

*   **FR1: Email Validation:** The system shall validate the email input field.
    *   **FR1.1:** The email validation shall ensure the presence of an '@' symbol.
    *   **FR1.2:** The email validation shall ensure a standard domain format (e.g., `example.com`).
*   **FR2: Password Validation:** The system shall validate the password input field.
    *   **FR2.1:** The password validation shall ensure a minimum length of 6 characters.

### Non-Functional Requirements

*   **NFR1: Responsiveness:** The login page shall be responsive, adapting its layout to various screen sizes and devices.
*   **NFR2: Theming:** The login form shall adhere to a sleek, dark-themed design.
*   **NFR3: Error Element ID (Email):** The error message element for email validation shall have the ID `emailError`.
*   **NFR4: Error Element ID (Password):** The error message element for password validation shall have the ID `passwordError`.
